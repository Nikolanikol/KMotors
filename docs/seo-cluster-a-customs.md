## 1. Диагноз — почему калькулятор на позиции 49–61

1. **`"use client"` — весь компонент клиентский.** Индексируемого HTML-текста почти нет: Google видит пустую оболочку. Для информационных запросов («растаможка», «цена») контента для ранжирования нет.
2. **Нет опорных статей.** Самый большой объём показов — не «калькулятор», а «растаможка / цена / стоимость авто из кореи». Эти запросы хотят **статью-гайд**, а инструмент — уже внутри неё.
3. **Нет lead-формы на калькуляторе.** Источник `car_calculator` в API есть, но компонент заявку не отправляет — CTA только ведёт в каталог. Посетитель с готовым расчётом уходит без захвата.
4. **Блог не ссылается на калькулятор** (1 пост из 52). Ноль внутреннего ссылочного веса.
5. **Кыргызстан не покрыт** (движки только RU/KZ/UZ), хотя спрос подтверждён: «растаможка авто из кореи в кыргызстан 2026» органически вышел на позицию 1.

**Два калькулятора — разные работы (не путать):**
- Калькулятор на карточке авто (`CustomsCalculator.tsx`) — низ воронки, у пользователя уже выбрана машина, форма `CarRequestForm` уже стоит. Не трогаем.
- Отдельный `/calculator` — SEO-лендинг под внешний трафик из Google (человек ещё НЕ выбрал авто). Лид ценнее («подбери под бюджет»). Его и прокачиваем.

## 3. Контент-план — весь контент живёт на `/calculator` (ru + en сразу)

Отдельной блог-статьи НЕТ (решение владельца). Вся текстовая часть — SSR-гайд на самой странице калькулятора, чтобы одна страница закрывала и INFO-, и TOOL-запросы.

### 3.1 SSR-гайд на /calculator
- **Target:** «растаможка авто из кореи», «цена/стоимость растаможки», «калькулятор растаможки», «растаможить авто из кореи» + гео-варианты.
- **Языки:** ru + en одновременно (тексты в словаре, как FAQ/META сейчас).
- **Объём текста:** 4000–6000 знаков SSR (сейчас индексируемого текста ~0).
- **Структура (серверный компонент, H1/H2 рендерятся на сервере):**
  1. H1 «Растаможка авто из Кореи 2026: рассчитать стоимость» + вводный абзац
  2. Из чего складывается стоимость (пошлина + утильсбор + таможенный сбор + НДС/акциз)
  3. Ставки 2026 для физлиц — таблицы по возрасту и объёму двигателя
  4. ⬇️ сам калькулятор (клиентский, как сейчас)
  5. Растаможка по странам: Россия / Казахстан / Узбекистан (Кыргызстан — позже)
  6. Доставка из Кореи — **вилка по маршрутам + «точную сумму считает менеджер»** (закрывает «калькулятор доставки», ведёт в форму)
  7. lead-форма + FAQ (FAQ уже есть в `calculator/page.tsx`)

### 3.2 Перелинковка (ссылочный вес)
- В 8 «быстрых» постах блога и в карточках каталога — контекстная ссылка на /calculator.
- Из /calculator — на релевантные сравнения моделей (genesis-gv70-vs-tucson и т.д.) и в /catalog.

## 4. Воронка привлечения (это и есть «инструмент привлечения»)

1. Пользователь считает растаможку на /calculator → видит итог.
2. Под итогом — **`CarRequestForm`** (тот же компонент, что на карточке авто): «Получить точный расчёт и подбор авто под этот бюджет».
3. `POST /api/telegram`, `source: "car_calculator"`, в `message` — параметры расчёта (страна, объём, возраст, итог) → менеджер получает лид с контекстом.
4. Существующий CTA → /catalog остаётся.

Лид с /calculator ценнее, чем с карточки: у человека ещё нет выбранной машины → это новый клиент на подбор.
